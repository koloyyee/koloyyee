## Why a homelab?

After college, my Dell Latitude sits idle. It runs Windows 11, and as a Unix user for decades, I prefer a Linux machine. I've been deploying web apps on remote VPS for years, yet I've never set up a server from scratch. Moreover, I recently began exploring DevOps, and I believe this will be a great opportunity to learn by doing.

## Goal
The goal is rather simple: I will try to replace Google Photos with [Immich](https://immich.app), build my own library with [Calibre-Web](https://github.com/janeczku/calibre-web), and dogfood my apps. Aside from privacy and dogfooding, I could apply my newly acquired knowledge in [Ansible](https://docs.ansible.com) and [Terraform](https://developer.hashicorp.com/terraform), creating a Debian server from scratch and use my network knowledge, how cool is that?!

## Challenges
Unlike traditional homelab where the "server" is always plugged into the Ethernet, I can only access the internet with Wi-Fi. That means I cannot use [Proxmox](https://www.proxmox.com/en/) and I don't want to pay for [Unraid](https://unraid.net/). At the same time I want to apply the DevOps knowledge, therefore I ruled out using [CasaOS](https://casaos.zimaspace.com/). Hence, I decided to go with a pretty bare-bones solution.

## Tech 
The server is a Dell Latitude, an Intel-based laptop with 512GB storage and 8GB RAM. Supported by Nginx as proxy server, Docker for running my apps in containers, ufw for firewall, [Tailscale](https://tailscale.com) for secure connectivity, [Ansible](https://docs.ansible.com) for configuration management, [Terraform](https://developer.hashicorp.com/terraform) for continuous deployment.

## Clean up
First of all, wiping out Windows then with a USB flash drive and a ISO file, I made Debian 13 as the server operating system. Follow by setting up `sudo`, installing essential packages including ufw, SSH, tailscale, Docker, Nginx, etc. After [setting up ufw](https://www.digitalocean.com/community/tutorials/how-to-set-up-a-firewall-with-ufw-on-ubuntu) and [SSH access](https://www.digitalocean.com/community/tutorials/how-to-use-ssh-to-connect-to-a-remote-server), the whole process took less than 2 hours — the server was ready to go!
